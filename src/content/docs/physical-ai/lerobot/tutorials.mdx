# SO-101로 Imitation Learning

이 튜토리얼에서는 LeRobot을 사용하여 실제 로봇에 모방 학습(Imitation Learning)을 적용하는 방법을 알아봅니다.

<Callout type="info" title="모방 학습이란?">
  모방 학습은 인간의 시연을 관찰하고 학습하여 로봇이 동일한 작업을 수행할 수 있도록 훈련하는 방법입니다. 
  로봇이 물체를 집거나, 조작하는 등의 복잡한 작업을 학습할 수 있습니다.
</Callout>

## 📋 준비물

### 필수 하드웨어
- **로봇 팔**: SO-100, SO-101, Koch v1.1 등 LeRobot 지원 하드웨어
- **원격 조작 장치**: 리더 암(leader arm) 또는 키보드
- **카메라**: 시각적 입력을 위한 웹캠 (선택사항이지만 권장)

### 소프트웨어 요구사항
- Python 3.8+
- PyTorch (CUDA 지원 권장)
- LeRobot 라이브러리

## 🚀 시작하기

### 1단계: 하드웨어 설정 및 캘리브레이션

먼저 로봇과 원격 조작 장치를 연결하고 캘리브레이션을 수행합니다.

```bash
# 로봇 연결 확인
python -m lerobot.scripts.find_devices

# 캘리브레이션 실행
python -m lerobot.scripts.calibrate \
  --robot-type so101 \
  --robot-device /dev/ttyUSB0
```

<Callout type="warning" title="디바이스 연결 주의">
  USB 디바이스의 경로는 시스템마다 다를 수 있습니다. `ls /dev/ttyUSB*` 명령으로 확인하세요.
</Callout>

### 2단계: 데이터 수집

이제 로봇을 원격 조작하여 작업을 시연하고 데이터를 수집합니다.

#### 녹화 시작

```bash
python -m lerobot.scripts.record \
  --robot-type so101 \
  --fps 30 \
  --repo-id ${HF_USER}/my_robot_dataset \
  --tags tutorial pick_place \
  --warmup-time-s 5 \
  --episode-time-s 30 \
  --reset-time-s 10
```

#### 데이터 수집 팁

<div style="background: var(--color-surface); padding: 1.5rem; border-radius: var(--radius-lg); margin: 1rem 0;">
  <h4 style="margin-top: 0;">📸 효과적인 데이터 수집을 위한 권장사항</h4>
  
  - **최소 50개 이상의 에피소드 녹화**: 모델이 일반화할 수 있도록 충분한 데이터 필요
  - **일관된 카메라 위치 유지**: 모든 녹화에서 동일한 시점 사용
  - **작업별 10개 에피소드**: 각 위치나 조건에서 최소 10회 시연
  - **점진적 난이도 증가**: 간단한 작업부터 시작하여 복잡도 증가
</div>

수집한 데이터로 신경망 모델을 훈련합니다.

### 4단계: 평가 및 실행

훈련된 모델을 로봇에서 실행하여 성능을 평가합니다.

```bash
# 훈련된 정책으로 로봇 제어
python -m lerobot.scripts.eval \
  --robot-type so101 \
  --policy-checkpoint outputs/train/act_my_robot/checkpoint_100.pth \
  --fps 30 \
  --episode-time-s 30
```

## 🎯 실전 예제: 물체 집기 작업

### 작업 설정
1. 테이블 위에 작은 물체(블록, 컵 등) 배치
2. 로봇이 물체를 집어 다른 위치로 이동
3. 다양한 시작 위치에서 반복

### 데이터 수집 프로토콜
```bash
# 5개의 다른 위치에서 각각 10회 시연
for position in {1..5}; do
  echo "Position $position - 10 episodes"
  python -m lerobot.scripts.record \
    --tags pick_place position_${position} \
    --num-episodes 10
done
```

## 🔧 문제 해결

### 일반적인 문제와 해결방법

<Callout type="warning" title="낮은 성공률">
  **문제**: 로봇이 작업을 제대로 수행하지 못함
  
  **해결책**:
  - 더 많은 데이터 수집 (100+ 에피소드)
  - 시연의 일관성 향상
  - 카메라 위치 재조정
</Callout>

<Callout type="danger" title="하드웨어 연결 오류">
  **문제**: 로봇이 인식되지 않음
  
  **해결책**:
  - USB 권한 확인: `sudo chmod 666 /dev/ttyUSB*`
  - 드라이버 재설치
  - 케이블 교체
</Callout>
