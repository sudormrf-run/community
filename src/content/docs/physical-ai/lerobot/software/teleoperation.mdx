## 리더-팔로워 방식 개요

리더-팔로워(Leader-Follower) 방식은 LeRobot에서 가장 직관적이고 효과적인 텔레오퍼레이션 방법입니다:

- **리더 로봇**: 사람이 직접 손으로 움직이는 로봇
- **팔로워 로봇**: 리더의 동작을 실시간으로 따라하는 로봇
- **장점**: 직관적이고 정확한 제어, 고품질 데이터 수집

<Callout type="tip" title="왜 리더-팔로워 방식인가?">
  리더-팔로워 방식은 로봇의 물리적 제약을 자연스럽게 반영하며, 인간의 직관적인 움직임을 그대로 전달할 수 있어 Imitation Learning에 최적의 데이터를 제공합니다.
</Callout>

## 하드웨어 준비

### 필요한 장비

1. **동일한 로봇 2대**: 리더용과 팔로워용
2. **컴퓨터**: 두 로봇을 동시에 제어할 수 있는 시스템
3. **USB 케이블**: 각 로봇 연결용
4. **카메라**: 데이터 수집용 (선택사항)

### 로봇 연결

```bash
# 연결된 로봇 확인
ls /dev/ttyUSB*
# 일반적으로 /dev/ttyUSB0 (리더), /dev/ttyUSB1 (팔로워)
```

## 리더-팔로워 설정

### 1단계: 환경 설정

```python
from lerobot.common.robot_devices.robots.factory import make_robot
from lerobot.common.robot_devices.motors.dynamixel import DynamixelMotorsBus

# 리더 로봇 초기화
leader = make_robot(
    robot_type="koch",  # 또는 사용하는 로봇 타입
    robot_kwargs={
        "leader_port": "/dev/ttyUSB0",
        "follower_port": "/dev/ttyUSB1",
    }
)
```

### 2단계: 캘리브레이션

리더와 팔로워가 정확히 동기화되도록 캘리브레이션합니다:

```python
# 로봇 캘리브레이션
print("리더 로봇을 홈 포지션으로 이동시켜주세요...")
leader.calibrate()

print("캘리브레이션 완료! 텔레오퍼레이션을 시작할 수 있습니다.")
```

### 3단계: 텔레오퍼레이션 시작

```python
# 리더-팔로워 모드 활성화
leader.connect()
leader.teleop_start()

print("리더 로봇을 움직이면 팔로워가 따라합니다.")
print("종료하려면 Ctrl+C를 누르세요.")

try:
    while True:
        # 리더의 위치를 읽어서 팔로워에 전달
        leader.teleop_step()
except KeyboardInterrupt:
    print("텔레오퍼레이션 종료")
    leader.disconnect()
```

## 데이터 수집

리더-팔로워 방식으로 고품질 학습 데이터를 수집합니다:

```python
from lerobot.common.datasets.lerobot_dataset import LeRobotDataset
from lerobot.common.robot_devices.cameras import OpenCVCamera

# 카메라 설정
cameras = {
    "top": OpenCVCamera(camera_index=0, fps=30),
    "wrist": OpenCVCamera(camera_index=1, fps=30)
}

# 데이터셋 생성
dataset = LeRobotDataset(
    repo_id="my_teleoperation_data",
    fps=30
)

# 데이터 수집 시작
episode_index = 0
while True:
    print(f"\n에피소드 {episode_index + 1} 시작")
    print("준비되면 Enter, 종료하려면 'q' 입력")
    
    user_input = input()
    if user_input.lower() == 'q':
        break
    
    # 에피소드 녹화
    leader.record_episode(
        dataset=dataset,
        episode_index=episode_index,
        cameras=cameras
    )
    
    episode_index += 1

print(f"총 {episode_index}개의 에피소드 수집 완료!")
```

## 안전 및 주의사항

### 속도 제한

```python
# 안전을 위한 속도 제한 설정
leader.set_max_speed(0.5)  # 최대 속도 50%로 제한
```

### 작업 공간 제한

```python
# 작업 공간 경계 설정
workspace_limits = {
    "x": (-0.3, 0.3),
    "y": (-0.3, 0.3),
    "z": (0.0, 0.4)
}
leader.set_workspace_limits(workspace_limits)
```

  - 로봇 주변에 충분한 공간을 확보하세요
  - 비상시 즉시 전원을 차단할 수 있도록 준비하세요
</Callout>

## 팁과 모범 사례

### 효과적인 데이터 수집

1. **다양한 시작 위치**: 매 에피소드마다 조금씩 다른 위치에서 시작
2. **속도 변화**: 빠른 동작과 느린 동작을 적절히 섞어서 수행
3. **실패 포함**: 완벽한 성공뿐만 아니라 실패 케이스도 수집
4. **반복 수행**: 같은 작업을 여러 번 반복하여 다양성 확보

### 문제 해결

<details>
<summary>리더와 팔로워가 동기화되지 않음</summary>

- 캘리브레이션 다시 수행
- USB 케이블 연결 확인
- 모터 ID 충돌 확인

</details>

<details>
<summary>동작이 부자연스러움</summary>

- 속도 제한 값 조정
- 모터 게인(gain) 값 조정
- 기계적 마찰 확인

</details>

## 다른 제어 방법

리더-팔로워 방식 외에도 다음과 같은 방법으로 로봇을 제어할 수 있습니다:

- **게임패드**: PlayStation/Xbox 컨트롤러 사용
- **키보드**: 키보드 단축키로 제어
- **SpaceMouse**: 3D 마우스로 정밀 제어

이러한 방법들은 리더-팔로워 방식보다 직관성은 떨어지지만, 특정 상황에서 유용할 수 있습니다.
